內部討論結論

歐巴「總部店」設計說明

新開一間獨立分店作為總部，串接專屬 LINE OA

1核心決策

太平店＝總部 → 新開「歐巴總部店」＝總部

總部店獨立串接 一個專屬 LINE OA，與分店分離。

2為什麼要獨立總部？

  • 會員資料集中：全台會員註冊後資料都會同步到總部店。若用太平店當總部，會員資料瀏覽不易。
  • 權益切割彈性：方便日後關閉總部時取消權益、或建立總部專屬權益。
    總部發行的權益 ＝ 組織通用。

3概念比喻：點數錢包

🏢 總部

總部錢包 ×5

只能新增總部權益錢包

🏪 分店

總部錢包 ×5 分店錢包 ×5

繼承總部 ＋ 可再新增自己的

4四種通知情境

關鍵：顧客要收到總部 LINE 推播，必須「加總部 LINE 好友 ＋ 綁定會員卡」。

# 情境 加總部LINE? 收到推播? 會員卡獲券?
1 A店加好友＋註冊，消費1,000升白金（總部入會禮＋等級禮） 否 ✗ 收不到 ✓ 仍獲得
2 A店加好友＋註冊，A店店員發一張票券（由 A 店發出） — ✓ 收到A店通知 ✓ 獲得
3 A店加好友＋註冊，年消60,000升鉑金（總部升等禮） 否 ✗ 收不到 ✓ 仍獲得
4 同上，但有加總部LINE＋綁定會員卡 是 ✓ 收總部推播 ✓ 獲得

💡 結論

要確保顧客一定收到「總部權益」的卡牌通知，必須：

  • ① 引導顧客加總部 LINE 好友並綁定會員卡
  • ② 確保總部官方帳號的訊息數足夠，才能發送大量通知

📌 備註：歐巴總部與凱惠的串接，預計找一家正式的凱惠分店處理。

5為何設計「總部通知」？＝ 節省訊息數

A店獲得總部權益 → A店通知＋總部通知 → 統一總部通知
A店消費使用權益 → A店通知＋總部通知 → 統一總部通知
A店獲得權益 → 到B店使用 → A店＋B店通知 → 統一總部通知

🎯 目的：避免訊息數用太多、減少打擾顧客 → 統一在總部通知。
